## Decisions made

- **Separate files for cover letter and README**: the cover letter is non-blind (for the editor), the README is blind (for reviewers).
  They do not share a file.
- **Skill does not commit or push**: follows project convention; the user handles git manually after reviewing the bundle.
- **Temporary `index-blind.qmd` for anonymization**: the skill never modifies `index.qmd` directly.
  If redactions are needed, a temporary blinded copy is rendered instead and deleted after bundle assembly.
- **Grant number and `bit.ly` URL stay in `index.qmd`**: they are the author's current source of truth.
  Redaction happens only in the blind copy for submission, not permanently in the source.
- **Retroactive fix of `legacy/submission-20260410/` deferred**: not part of this session.
  The recommended fix is to delete the current bundle and regenerate with the new skill once it has been smoke-tested.
